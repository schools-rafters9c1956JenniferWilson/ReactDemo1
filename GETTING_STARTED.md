## 💡 学习技巧

### ✅ 必做的事

1. **边学边做** - 不要只看，一定要手工输入和修改代码
2. **测试假设** - 改变代码看看会发生什么
3. **对比学习** - 使用快速参考对比 Vue 和 React
4. **查看官方文档** - 当你想深入某个概念时
5. **做笔记** - 记录关键概念和容易混淆的地方

### ❌ 避免的事

1. **只阅读，不操作** - 这样学不会
2. **跳过基础直接学高级** - 会很困惑
3. **忽视依赖数组** - 这是常见错误之源
4. **直接修改状态对象** - 会导致难以追踪的 bug
5. **用索引作为列表的 key** - 会导致数据混乱

## 🎯 关键概念清单

在继续之前，你应该理解：

- [ ] **JSX** 是 JavaScript 的扩展
- [ ] **useState** 用来管理组件状态
- [ ] **useEffect** 用来处理副作用
- [ ] **Props** 是从父组件传递数据到子组件
- [ ] **单向数据流** - React 强制数据从上到下流动
- [ ] **依赖数组** 控制 effect 何时运行
- [ ] **组件化** - 将 UI 拆分成独立的、可复用的部分
- [ ] **性能优化** - 避免不必要的重新渲染

## ❓ 常见问题

### Q: 我怎么开始学习？

**A**:

1. 确保 `npm run dev` 正在运行
2. 打开浏览器访问 `http://localhost:5173/`
3. 按顺序学习课程
4. 每学完一课就修改代码进行实验

### Q: 我需要会 TypeScript 吗？

**A**: 不需要，但这个项目使用了 TypeScript。如果你不熟悉，可以：

- 先学习 JavaScript 的基础
- TypeScript 的语法很简单，类型注解就像在 JSDoc 中写注释
- 本项目中的 TypeScript 用法很基础，容易理解

### Q: 为什么我的代码不工作？

**A**: 常见原因：

1. 忘记了依赖数组 - 查看 REACT_LEARNING_SUMMARY.md
2. 直接修改了状态 - 需要使用 setState
3. 忘记了 return JSX - 每个组件都需要返回 JSX
4. Props 拼写错误 - 检查组件中的 Props 名称

### Q: 我能修改学习中心的代码吗？

**A**: 当然可以！这是学习的一部分：

1. 打开对应的 `.tsx` 文件
2. 修改代码
3. 保存 - 浏览器会自动更新
4. 测试修改的效果

### Q: 如何使用快速参考？

**A**:

1. 在学习中心左侧选择「快速参考」
2. 或者打开 `REACT_GUIDE.md` 和 `CODE_SNIPPETS.md` 在编辑器中查看
3. 查找你要的模式，复制代码
4. 根据你的需求修改
